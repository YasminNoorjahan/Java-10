Why Java 10?

Java 10 was introduced to align with a new release cadence that emphasizes predictability, faster delivery of new features, and increased community involvement. The goal is to make Java development more agile, responsive, and adaptive to the changing requirements of the software industry.

Java 10, released in March 2018, was part of Oracle's new release cadence, which introduced a time-driven model for Java releases.

New Features in Java 10

Local Variable Type-Inference

var is used to remove boilerplate code, making your code more concise and readable. 

        
            Map<Integer, Student2> studentMap = new HashMap<>();
            // Adding students to the HashMap
            studentMap.put(1, new Student2(1, "Alice", 85.5));
            studentMap.put(2, new Student2(2, "Bob", 92.0));
            studentMap.put(3, new Student2(3, "Charlie", 78.3));
            studentMap.put(4, new Student2(4, "David", 94.8));

            studentMap.entrySet().stream().forEach(entry -> System.out.println(entry.getValue()));

            // What type of variable we can assign to the below a map or a student object?
            studentMap.entrySet().stream().max((stu1, stu2) -> stu1.getValue().getMarks() > stu2.getValue().getMarks() ? 0 : 1);

            // If we can't predict the value type, then we can use 'var'
            var student = studentMap.entrySet().stream().max((stu1, stu2) -> stu1.getValue().getMarks() > stu2.getValue().getMarks() ? 0 : 1);

The compiler infers the type from the type of the initializer present on the right-hand side. And this is only for local variables, not for member variables.

Finally, note that there is no runtime overhead in using var nor does it make Java a dynamically typed language. The type of the variable is still inferred at compile time and cannot be changed later.
